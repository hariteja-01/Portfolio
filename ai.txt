================================================================================
                     HARI TEJA PATNALA — PORTFOLIO WEBSITE
                         Complete Technical Description
================================================================================


1. OVERVIEW
--------------------------------------------------------------------------------

This is a premium, interactive, single-page portfolio website built for
Hari Teja Patnala — an Algorithm Engineer & Full-Stack Developer. The site
showcases three flagship projects through an immersive, three-view architecture
that transitions users from a landing page into full project deep-dives using
scroll-driven video expansions and canvas-based frame-by-frame scroll
animations. It is designed to feel like a high-end product page (think Apple)
rather than a typical developer portfolio.

URL Structure: Single page (/) — all navigation is client-side with animated
view transitions.


2. TECH STACK
--------------------------------------------------------------------------------

  Framework        :  Next.js 14.2 (App Router, static export support)
  Language         :  TypeScript (strict)
  UI Library       :  React 18.3
  Styling          :  Tailwind CSS 3.4 + CSS custom properties (CSS variables)
  Animation        :  Framer Motion 11.15 (page transitions, scroll-linked
                      transforms, spring physics, AnimatePresence)
                      GSAP 3.14 (available for complex timelines)
  Icons            :  Lucide React 0.575
  Utility          :  clsx + tailwind-merge (conditional class composition)
  Typography       :  Google Fonts — Inter (loaded via next/font)
  Build            :  PostCSS + Autoprefixer
  Linting          :  ESLint with eslint-config-next
  Package Manager  :  npm
  Dev Server       :  next dev


3. COLOR SYSTEM & THEMING
--------------------------------------------------------------------------------

The site supports full DARK/LIGHT mode toggling, persisted to localStorage.
Theme switching is handled by a React Context (ThemeProvider) that dynamically
sets CSS custom properties on <html>.

  Dark Mode (default):
    --bg-primary      : #0A0A0F  (near-black)
    --bg-secondary    : #111118

4. WEBSITE ARCHITECTURE — THREE-VIEW SYSTEM
--------------------------------------------------------------------------------

The entire site is a single React page (app/page.tsx) that manages three
mutually exclusive "view states", animated with Framer Motion's AnimatePresence:

  ┌──────────────────────────────────────────────────────────────────┐
  │  VIEW 1: "landing"                                              │
  │  ─────────────────                                              │
  │  The main portfolio landing page. Contains:                     │
  │   • HeroSection (full-screen intro with 3D character)           │

  │   • ProjectCardStack (interactive fan-out card carousel)        │
  │   • ContactSection (contact form + social links)                │
  │   • Footer                                                      │
  │  Each section separated by animated gradient dividers.          │
  │                                                                  │
  │  USER ACTION: Click a project card →                            │
  └──────────────────────┬───────────────────────────────────────────┘
                         │
                         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │  VIEW 2: "expanding"                                            │
  │  ──────────────────                                             │
  │  A full-screen scroll-to-expand video transition:               │
  │   • Background image from the project                           │
  │   • A small centered video frame that EXPANDS as you scroll     │
  │   • Title & date overlay with mix-blend-mode text               │
  │   • "Scroll to expand" indicator                                │
  │   • Once fully expanded: shows project summary + CTA buttons    │
  │     ("Enter Full Experience →" and "← Back")                    │
  │                                                                  │
  │  Uses: ScrollExpandMedia component (custom scroll physics,      │
  │         wheel + touch + native scroll support)                  │
  │                                                                  │
  │  USER ACTION: Click "Enter Full Experience →" →                 │
  └──────────────────────┬───────────────────────────────────────────┘

                         ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │  VIEW 3: "deepdive"                                             │
  │  ─────────────────                                              │
  │  The full project showcase with:                                │

5. COMPONENT-BY-COMPONENT BREAKDOWN
--------------------------------------------------------------------------------

5.1 LoadingScreen
    Location  : components/LoadingScreen.tsx
    Purpose   : Full-screen branded intro that appears for ~2 seconds on
                initial page load.
    Visuals   : Monospace text "<HT/>" with each letter animating in
                sequentially (staggered fade-up). Below it, a gradient
                progress bar (cyan → violet → orange) scales in from left.
                The entire screen then fades out to reveal the site.

5.2 CustomCursor
    Location  : components/CustomCursor.tsx
    Purpose   : Replaces the default OS cursor on desktop devices (pointer:fine)
                with a custom animated cursor.
    Visuals   : Three layers —
                • Inner Dot: 8px cyan (#00F0FF) circle, follows mouse instantly
                • Outer Ring: 36px circle border that follows with a smooth
                  lerp (0.12 factor), creating a trailing effect
                • Glow: Full-screen radial gradient that follows the mouse,
                  casting a subtle cyan glow on surrounding content
    States    :
      - default  : 8px dot + 36px ring
      - pointer  : dot disappears, ring grows to 50px with fill
      - project  : dot disappears, ring grows to 80px, shows "View →" text
      - wave     : ring 60px, shows 👋 emoji
    Note: Disabled on mobile/touch devices.

5.3 ScrollProgress
    Location  : components/ScrollProgress.tsx
    Purpose   : A thin progress bar at the very top of the viewport showing
                how far the user has scrolled on the page.

5.4 Navbar
    Location  : components/Navbar.tsx (311 lines)
    Purpose   : Persistent top navigation bar.
    Features  :
      - Glassmorphic background (blur + transparency)
      - Hides on scroll-down, reappears on scroll-up
      - Left: Brand mark "HT" with gradient
      - Center: Navigation links (About, Projects, Contact) — only in
        landing view
      - Right: Dark/Light theme toggle button + Resume download button
      - In project view: shows "← Portfolio" back button instead of nav links
      - Mobile: Full-screen hamburger menu with staggered animations
      - All hover states have smooth color transitions

5.5 HeroSection
    Location  : components/sections/HeroSection.tsx (302 lines)
    Purpose   : Full-screen hero/intro section — the first thing visitors see.
    Layout    : Two-column grid on desktop (3/5 + 2/5 split):
      LEFT COLUMN:
        - "Available for opportunities" badge (pulsing green dot + glass card)
        - Name "Hari Teja / Patnala" — each letter animates in individually
          with staggered delays (character-by-character reveal)
        - Gradient underline bar (cyan → violet → orange) scales in
        - Tagline: "Algorithm Engineer & Full-Stack Developer"
        - Description paragraph with color-highlighted keywords:
          "800+ LeetCode" (cyan), "Oracle Cloud certified" (violet),
          "algorithms, ML" (orange)
        - Two CTA buttons:
          • "Explore My Work" — gradient filled, rounded pill with glow shadow
          • "Download CV" — outlined pill with hover glow effect
        - Social links row: GitHub, LinkedIn, LeetCode, Email — each with
          underline-on-hover animation
      RIGHT COLUMN (desktop only):
        - Student3D component — an interactive 3D-ish animated student
          character

    Background layers:
      1. SparkleBackground — full-screen canvas particle system
      2. Four animated gradient orbs (blurred, floating, 25-30s cycles):
         - Cyan orb (600px, at top-left)
         - Violet orb (500px, at center-right)
         - Orange orb (450px, at bottom-center)
         - Green orb (300px, at center)
      3. Dot grid pattern overlay (30px spacing, very subtle)
      4. Scroll indicator arrow at bottom ("↓ Scroll to explore")

5.6 SparkleBackground
    Location  : components/SparkleBackground.tsx (206 lines)
    Purpose   : A canvas-based particle system covering the entire hero section.
    Visuals   : ~80 diamond/cross-shaped sparkle particles in 6 colors (cyan,
                violet, orange, gold, green, pink). Each particle:
      - Drifts upward slightly with random velocity
      - Has a pulsing shimmer opacity animation
      - Gets gently attracted toward the mouse cursor (within 200px radius)
      - Has a soft glow (canvas shadowBlur)
      - Spawns new burst particles near the mouse position every 8 frames
    Shape     : 4-point star + rotated cross (drawn with canvas paths)
    Performance: Bounded at count + 20 max particles; uses requestAnimationFrame

5.7 Student3D
    Location  : components/Student3D.tsx (324 lines)
    Purpose   : An interactive CSS-rendered 3D student character displayed
                next to the hero text.
    Features  :
      - Responds to scroll with head-tilt animation
      - Has a "wave" interaction (👋 cursor state)
      - Built with layered div transforms for pseudo-3D appearance
      - AnimatePresence for state transitions

5.8 AboutSection

      - Education info: LPU, B.Tech CSE, CGPA 9.18/10, expected Jul 2027

5.9 GlassmorphicCard
    Location  : components/GlassmorphicCard.tsx (50 lines)
    Purpose   : Reusable glassmorphism container card.
    Visuals   : Semi-transparent background + 20px blur + 180% saturation +
                subtle border + rounded corners (16px). Optional hover glow
                with configurable color (defaults to cyan) + lift effect.

5.10 ProjectCardStack
    Location  : components/sections/ProjectCardStack.tsx (292 lines)
    Purpose   : Displays all three projects as an interactive "fan-out"
                card stack carousel.
    Features  :
      - Uses the CardStack UI component (fan-spread layout with 3D perspective)
      - Each card shows:
        • Project cover image (from /images/<project>/1.jpg)
        • Auto-playing hover video (mp4) with poster fallback (webp)
        • Project name, subtitle, role
        • Tech stack tags (colored pills)
        • Stats row (e.g., "Learning Boost: 40%", "Algorithms: 15+")
        • "Explore Project →" button styled with project's themeColor
      - Cards spread in a fan layout with configurable overlap, tilt, depth
      - Keyboard navigation (left/right arrows)
      - Drag-to-navigate on touch
      - Dot indicators below the stack
      - Spring-based physics animations (stiffness: 280, damping: 28)

5.11 CardStack (UI Primitive)
    Location  : components/ui/card-stack.tsx (412 lines)
    Purpose   : Generic, reusable fan-spread card carousel with 3D perspective.
    Config    :
      - cardWidth: 520px, cardHeight: 320px
      - overlap: 0.48, spreadDeg: 48°
      - perspectivePx: 1100, depthPx: 140, tiltXDeg: 12°
      - activeLiftPx: 22
      - Loop mode, auto-advance optional
      - Drag-to-dismiss with velocity detection
    Animation : Framer Motion spring transitions for 3D rotation, translation,
                scale, and opacity.

5.12 ScrollExpandMedia
    Location  : components/ui/scroll-expansion-hero.tsx (316 lines)
    Purpose   : The immersive scroll-driven video expansion transition
                (VIEW 2: "expanding").
    How it works:
      - On entering this view, a background image fills the viewport
      - A small centered frame (video or image) starts at ~40% size
      - As the user scrolls/swipes, the frame smoothly expands toward
        fullscreen using accumulated scroll delta
      - Title (project name) and date (subtitle) overlay with optional
        mix-blend-mode "difference" for text blending
      - An animated "Scroll to expand" indicator pulses at the bottom
      - Once expansion reaches 100%, children content is revealed below
        with a fade-in animation
      - Supports: wheel events, touch events, native scroll fallback
      - Mobile-responsive: adjusts initial size and uses touch gestures

5.13 CanvasScrollAnimation
    Location  : components/CanvasScrollAnimation.tsx (195 lines)
    Purpose   : Frame-by-frame image sequence animation driven by scroll
                position (Apple-style product page effect).
    How it works:
      - Pre-loads 200 JPEG frames from /images/<project>/1.jpg to 200.jpg
      - Uses an HTML5 <canvas> element (sticky position, 100vh)
      - The wrapper div is 500vh tall — as the user scrolls through it,
        Framer Motion's useScroll tracks progress (0.0 → 1.0)
      - progress * 200 = current frame index
      - Each frame is drawn with object-fit: cover calculations
      - Shows a loading progress bar (themed) while images load
    Per project: 200 frames each (600 total frames across all 3 projects)

5.14 ScrollTextOverlays
    Location  : components/ScrollTextOverlays.tsx (134 lines)
    Purpose   : Text sections that fade in and out over the canvas scroll
                animation at specific scroll positions.
    Structure : 4 text sections per project, each with title + subtitle:
      - Section 1 (0%–15%): Project name + tagline
      - Section 2 (25%–40%): Key feature 1
      - Section 3 (50%–65%): Key feature 2
      - Section 4 (75%–90%): Project ethos statement
      - Role tag at top (e.g., "HARI TEJA PATNALA — Algorithm Engineer")
    All text: large (5xl–8xl), bold, white, with text-shadow for contrast
              against the video frames.
